Recordatorios operativos

Antes del simulacro
  • Conectar servidor por cable Ethernet si es posible y mantener cargador activo.
  • Confirmar IP LAN activa y acceso desde otro PC a /simulador.
  • Verificar /health y /health/ready en la IP LAN.
  • No ejecutar importaciones, backups ni tareas pesadas de IA antes de iniciar.
  • Validar red privada y firewall con puerto 4000 habilitado.
  • Realizar prueba escalonada 5 -> 10 -> 25 -> 50 estudiantes.
Durante el simulacro
  • Monitorear healthchecks y logs solo ante incidencias.
  • No reiniciar Docker salvo emergencia real.
  • No ejecutar reset, prepare ni importacion de colegios.
  • Si falla un estudiante: recargar navegador y validar red local antes de cambios mayores.
  • Si fallan varios equipos: revisar WiFi/LAN primero, luego revisar API.
Despues del simulacro
  • Revisar resultados y reportes finales por rol.
  • Guardar evidencia y registrar incidentes observados.
  • Ejecutar backup posterior si los datos deben conservarse.
  • No borrar datos sin respaldo y aprobacion operativa.

Problemas comunes y respuesta rapida
  • Otro PC no abre la app: validar misma red, IP, firewall y puerto 4000.
  • Estudiante no inicia prueba: validar sesion, prueba asignada y logs API.
  • Sistema lento: evitar IA pesada y revisar calidad de red antes de reiniciar.
  • Error 500: registrar hora e inspeccionar docker compose logs api --tail=100.
